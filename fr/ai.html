IA & Agents

Je travaille concrètement avec les LLM — agents de code locaux, plugins et skills Claude Code, serveurs MCP, et l'expérience occasionnelle de casser mes propres modèles. Cette page suit ce que je livre, ce que je lis et ce que je démonte.

Focus Actuel

  • —Agents de code locaux (Ollama, Gemma, boucles ReAct)
  • —Plugins et skills Claude Code maison
  • —Serveurs Model Context Protocol (MCP) pour de vrais workflows de dev
  • —Exemples adversariaux et robustesse des modèles en NLP
  • —Inférence native dans le navigateur avec ONNX Runtime

Outils & Expériences

nip

[Open Source · Agent ReAct]

Un agent CLI local de code propulsé par Gemma sur Ollama. Implémente une boucle ReAct avec tool-calling pour lire, écrire et refactoriser du code sur votre machine, sans appels au cloud ni vendor lock-in.

GitHubnorth_east

claude-human-review

[Plugin Claude Code]

Un plugin Claude Code qui raconte chaque édition faite par l'agent en clair, pour que vous puissiez approuver ou annuler avec tout le contexte. Né de la méfiance envers les diffs silencieux — du human-in-the-loop sans friction.

GitHubnorth_east

article-writing-skills

[Claude Skills · Bibliothèque de Prompts]

Une collection de prompts d'écriture et de skills Claude modélisés sur les voix d'ingénieurs et de chercheurs connus. Fonctionne avec Claude Code, ChatGPT, Gemini et n'importe quel LLM — un terrain de test pour voir jusqu'où peut aller le transfert de style par prompt.

GitHubnorth_east

MERN MCP

[Serveur MCP]

Un serveur Model Context Protocol open-source qui automatise le scaffolding CRUD full-stack pour les apps MERN — des modèles Mongoose aux composants React — avec un workflow d'aperçu-avant-application pour garder l'agent en laisse.

GitHubnorth_east

BG Remover AI

[Navigateur · ONNX Runtime]

Un outil rapide, privé et sans serveur pour supprimer les arrière-plans, qui s'exécute entièrement dans le navigateur via des modèles ONNX. Pas d'upload, pas de tracking — explorant jusqu'où on peut pousser le ML côté client en 2026.
